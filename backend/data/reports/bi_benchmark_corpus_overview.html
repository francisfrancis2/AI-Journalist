3.1 Per-document structural features

The app asks Claude Haiku to extract these fields from each BI transcript:

  • hook_type: for example stat, question, scene, or claim.
  • hook_text: the first 2-3 sentences or opening hook pattern.
  • act_count, act_titles, and act_durations_seconds: structure and pacing.
  • has_human_story and human_story_act: whether and where a human element appears.
  • closing_device: for example forward look, open question, summary, or call to action.
  • stat_count: number of specific facts, figures, dollar amounts, percentages, or numerical claims.
  • rhetorical_question_count: question-driven narration style.
  • title_formula: for example "how X became Y", "why X is Z", "the rise of", or other.

3.2 Cross-corpus synthesized pattern library

After extracting document-level structures, Claude Sonnet synthesizes them into the active BIPatternLibrary:

  • Average act count.
  • Average act duration.
  • Hook type distribution.
  • Title formula distribution.
  • Closing device distribution.
  • Average statistic count.
  • Average rhetorical-question count.
  • Average act where the human story appears.
  • Up to five sample hooks.
  • All reference titles.

4. How Corpus Build Works

  1. Fetch candidate videos: CorpusBuilderAgent asks YouTubeFetcher for Business Insider channel videos, requesting more than the target number so filtering can remove unsuitable items.
  2. Filter for documentary length: only videos between 300 and 1200 seconds are considered.
  3. Skip duplicates: videos already present in bi_reference_docs are skipped.
  4. Fetch transcripts: English transcripts are fetched in batches with concurrency of 5.
  5. Extract structure: each transcript is passed to Claude Haiku using the document-structure schema.
  6. Save reference docs: each processed BI video is stored in bi_reference_docs with metadata, transcript, and extracted structure.
  7. Synthesize patterns: all structures are passed to Claude Sonnet to produce the pattern library.
  8. Persist library: the synthesized library is saved to bi_pattern_library and written to backend/data/bi_patterns.json.

The build can be run from the original script path python -m backend.scripts.build_corpus, or from the newer admin route POST /api/v1/benchmarks/rebuild, which the Benchmarking Admin UI calls.

5. How Evaluation And Benchmarking Are Used In The Pipeline

5.1 Pipeline order

The LangGraph pipeline runs in this order:

researcher -> analyst -> storyline_creator -> evaluator + benchmarker -> scriptwriter -> script_evaluator

The editorial evaluator and BI benchmarker run in parallel inside the evaluator graph node. The script evaluator runs after the final script has been written.

5.2 Editorial Evaluator

The editorial evaluator does not depend on the Business Insider corpus. It evaluates the selected storyline against professional editorial standards and decides whether the storyline is ready for scripting.

CriterionWeightQuestion being answered
Factual accuracy25%Are claims well-sourced and verifiable?
Narrative coherence20%Does the story flow logically?
Audience engagement20%Will viewers stay engaged for 10-15 minutes?
Source diversity15%Are multiple perspectives and source types represented?
Originality10%Does the story have a fresh angle?
Production feasibility10%Can the video realistically be produced?

A combined score of 0.75 or higher approves the storyline for scripting. Below that threshold, the graph can refine the storyline or request more research, depending on the state and configured cycle limits.

5.3 BI Benchmark Agent

The BI benchmarker compares the selected storyline against the active BI pattern library. It currently scores the storyline before the final script is generated. It does not gate the pipeline by itself; it adds benchmark metadata, strengths, gaps, grade, and similarity score for product feedback and later analysis.

BI benchmark criterionWeightWhat it checks
Hook potency20%Whether the opening resembles BI's statistic, dramatic moment, or counter-intuitive claim style.
Title formula fit10%Whether the title matches patterns like "How X became Y", "Why X is Z", "The rise/fall of X", "Inside X", or "X explained".
Act architecture20%Whether act count and act duration fit the synthesized BI average.
Data density15%How many specific stats, figures, and numerical claims appear in the storyline.
Human narrative placement15%Whether a human story appears, typically around acts 4-5.
Tension-release rhythm10%Whether the storyline alternates tension, context, human stakes, and resolution.
Closing device10%Whether the ending resembles BI's forward-looking closing style.

Grades are computed from the weighted score: A at 0.85 or above, B at 0.70 or above, C at 0.55 or above, and D below 0.55.

5.4 Post-script Script Evaluator

The post-script evaluator audits the final generated script. It uses the final narration, section structure, source references, prior editorial evaluation, prior benchmark report, and BI benchmark context when available.

Its output includes an overall grade, production-readiness flag, strengths, weaknesses, rewrite priorities, section-by-section audit notes, and a best-in-class comparison block when a valid benchmark context exists.

6. Current Benchmarking Reliability

Current reliability level: not reliable for BI comparison.

The active library object exists, but its doc_count is zero and every pattern field is empty or zero. Any BI similarity score generated from this state would be based on an empty benchmark context rather than actual BI reference material.

A useful detail from the code review: load_active_benchmark_library can return a library object even when doc_count = 0. The service adds a note saying the corpus is below the recommended minimum of 20 documents, but the benchmarker currently only checks whether a library object exists. That means an empty library can still reach the LLM benchmark prompt.

Recommended product behavior is stricter: treat doc_count < bi_corpus_min_docs as not ready for scoring, skip BI benchmark generation, and show an explicit admin action to rebuild the BI corpus.
